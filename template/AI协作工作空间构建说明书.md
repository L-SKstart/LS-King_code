# 🤖 AI 协作工作空间 — 构建说明书

> **本文档供 AI 阅读** — 描述一套经过实战验证的工作空间体系，
> 其他 AI 读完本文档即可自行构建相同或类似的工作空间。

---

## 一、这是什么

一个 **人和AI 协作的工作空间**，核心是给 AI 定规矩、管协作、留记录。  
不需要复杂的工具链，只需要：

- 一个文件夹（工作空间）
- 几个 Markdown 文件
- 一个或多个 AI 编码助手

---

## 二、目录结构

```
[根目录]/
├── 工作规范.md           # 规则体系 — AI 的行为准则（核心）
├── memory.md             # 记忆文件 — 团队共享的上下文（唯一）
├── 索引.md               # 导航索引 — 所有文件的入口
├── chat.md               # 沟通通道 — 人和AI、AI和AI的协作记录
├── 违规记录.md           # 违规追踪 — 记录谁的哪些违规
├── 需求要求记录.md       # 需求管理 — 用户的需求和待办
├── docs/
│   ├── manuals/          # 操作手册（步骤≥3的操作）
│   ├── references/       # 参考文档
│   └── archive/chat/     # 聊天归档
├── assets/               # 资源文件（截图/设计素材/大文件）
├── scripts/              # 脚本（shell/sql/python）
└── tools/                # AI 客户端工具
```

**根目录原则：** 只放每天必用的文件（6个核心 md + 配置文件）。非高频的放子目录。

## 四、核心机制

### 4.1 规则体系（工作规范.md）

两条至高规则 + 22条硬性规则，用优先级分层的结构：

| 层级 | 说明 | 示例 |
|:----:|------|------|
| 🔴 第零条 | 凌驾一切 — 每次回复前必读规范、必打招呼 | "您好，[名字]，[AI名] 就位" |
| 🔴 第零条之二 | 🧠🎯🔧 三角色检查 | 影响协作？出错看到什么？可回滚？ |
| 🔴 红线规则 | 漏一次记一次违规 | 规则8同步chat、规则10打招呼、规则18待办清单、规则20推送 |
| 🟡 普通规则 | 违反须记录但不立即惩罚 | 文件标注、中文注释等 |

**关键设计原则：** 规则是给 AI 读的，不是给人读的。每条规则简洁到 AI 能瞬间理解。

### 4.2 协作机制

```
用户提需求 → AI 打招呼+对时 → 读规范/记忆/chat → 执行 → 更新chat.md
```

**沟通格式（chat.md）：**

```markdown
### 🤖 AI名称：[MM-DD HH:MM] 做了什么

@其他AI ✅ 说明...

| 文件 | 变更 |
|------|------|
| xxx.md | 改了什么 |

⏱️ MM-DD HH:MM
---
```

**关键规则：**

- 只追加末尾，不插入中间
- 每条消息带时间戳 + 主题摘要
- 操作后必须 @ 所有同事

### 4.3 违规机制

```
违规记录 → 累计次数 → 1-2次警告 → 3-4次严重警告 → ≥5次惩罚
```

- AI 可自报违规，也可被其他 AI 发现后记录
- 每次违规记录包含：日期、涉及规则、违规内容、处理

### 4.4 记忆机制

- **memory.md 是唯一记忆文件**，禁止新建其他记忆文件
- 所有路径、服务器信息、技术上下文统一存于此
- 新增信息只能追加到 memory.md，不能新建文件

## 五、关键规则速查（复用时必须保留的）

| 条款 | 内容 | 为什么重要 |
|:----:|------|-----------|
| 第零条 | 每次回复前读chat+规范，打招呼+自报+对时 | 让AI不脱离上下文工作 |
| 第零条之二 | 三角色检查 | 让AI考虑全局而非只答问题 |
| 规则8 | 操作后同步chat.md，只追加末尾 | 团队协作的基础 |
| 规则10 | 每回复必打招呼+自报+对时 | 让用户感知AI在线 |
| 规则15 | memory.md为唯一记忆文件 | 防止记忆分散 |
| 规则18 | 末尾待办清单 | 让用户知道下一步 |
| 规则19 | 新增文件更新索引 | 文件可发现 |
| 规则20 | Git推送规则 | 分支管理 |

## 六、给你的 AI 的初始化提示词

当你要让一个新的 AI 接入这套体系时，先给它看 `工作规范.md`，然后告诉它：

> "你现在加入了一个 AI 协作工作空间。请先读取 `工作规范.md` 了解规则，
> 再读 `memory.md` 了解上下文，然后读 `chat.md` 看最新进展。
> 每次回复以'您好，[用户名]，[AI名] 就位'开头。
> 操作完成后同步到 chat.md，@ 其他同事。"

## 七、这套体系解决的核心问题

| 问题 | 这套怎么做 |
|------|-----------|
| AI 不知道边界 | 22条规则明确什么能做、什么不能做 |
| 多AI沟通混乱 | chat.md 统一通道 + 格式规范 |
| 出了错不知道 | 违规记录 + 三角角色检查 |
| 找不到文件 | 索引.md 统一导航 + 目录标准化 |
| 记忆丢失 | memory.md 唯一记忆，所有AI共用 |
| 操作不可追溯 | chat.md 时间线 + 文件修改标注 |
