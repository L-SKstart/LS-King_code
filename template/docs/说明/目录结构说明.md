# 目录结构说明 — 文件该放哪

> 一套标准化的开发工作空间目录结构

---

```
[你的工作空间根目录]/
│
├── 📄 工作规范.md          # 🔴 核心 — AI 规则体系（22条+2条至高）
├── 📄 memory.md             # 🔴 核心 — 团队记忆（路径/服务器/技术上下文）
├── 📄 索引.md               # 🔴 核心 — 文件导航索引
├── 📄 违规记录.md           # 违规追踪
├── 📄 需求要求记录.md       # 需求管理
├── 📄 chat.md               # 团队沟通通道
│
├── 📁 docs/                 # 所有文档
│   ├── 📁 manuals/          # 操作手册（步骤≥3的操作）
│   ├── 📁 references/       # 参考文档（规范以外的参考资料）
│   └── 📁 archive/chat/     # 聊天记录归档
│
├── 📁 assets/               # 资源文件
│   ├── 📁 screenshots/      # 截图
│   ├── 📁 logos/            # 设计素材/图标
│   └── 📁 downloads/        # 下载的大文件
│
├── 📁 scripts/              # 脚本
│   ├── 📁 shell/            # Shell 脚本
│   ├── 📁 sql/              # SQL 脚本
│   └── 📁 python/           # Python 脚本（如有）
│
├── 📁 tools/                # 工具客户端
├── 📁 skills/               # AI 技能
│
├── 📄 reasonix.toml         # Reasonix 配置文件（可选）
├── 📄 .gitignore            # Git 忽略规则
└── 📄 .env                  # 环境变量（凭据，已 gitignore）
```

## 各目录用途

| 目录 | 放什么 | 什么时候用 |
|------|--------|-----------|
| `根目录` | 只有每天必看的核心文件 | 每次打开工作空间 |
| `docs/manuals/` | 操作手册（≥3步的操作） | 遇到复杂操作时翻看 |
| `docs/references/` | 参考资料（宪法/团队分工/错误记录） | 需要查信息时 |
| `docs/archive/chat/` | 历史聊天记录 | chat.md超1000行时归档 |
| `assets/screenshots/` | 截图/图片 | 发图给AI时 |
| `assets/logos/` | Logo/设计素材 | 品牌相关 |
| `scripts/shell/` | Shell 脚本 | 执行自动化任务 |
| `scripts/sql/` | SQL 脚本 | 操作数据库 |

## 核心原则

1. **根目录只放日常必用的文件**（>1次/天）
2. **操作手册放 docs/manuals/**（偶尔才看）
3. **参考文档放 docs/references/**（需要时才查）
4. **资源文件放 assets/**（按类型细分）
5. **新增文件后更新 索引.md**
