# 快速上手指南 — 从0到1搭建 AI 协作工作空间

> ⏱️ 预计用时：15分钟

---

## 第1步：创建目录结构

在工作空间根目录（如 `D:\MyWorkspace\`）创建：

```bash
mkdir -p docs/manuals docs/references docs/archive/chat
mkdir -p assets/screenshots assets/logos assets/downloads
mkdir -p scripts/shell scripts/sql
```

## 第2步：替换占位符

打开所有 `TEMPLATE_*.md` 文件，批量替换以下内容：

| 占位符 | 替换为（示例） |
|--------|-------------|
| `[你的名字]` | `张三` |
| `[你的工作空间路径]` | `D:\MyWorkspace` |
| `[你的AI名称]` | `Copilot` / `Reasonix` 等 |
| `[你的团队名]` | `我的团队` |
| `[你的服务器IP:端口]` | `192.168.1.100:22` |
| `[你的数据库信息]` | （按需） |

## 第3步：把模板文件复制到工作空间

```bash
# 模板文件 → 工作空间根目录
cp TEMPLATE_工作规范.md  [你的工作空间]/工作规范.md
cp TEMPLATE_memory.md    [你的工作空间]/memory.md
cp TEMPLATE_索引.md      [你的工作空间]/索引.md
cp TEMPLATE_违规记录.md  [你的工作空间]/违规记录.md
cp TEMPLATE_需求要求记录.md  [你的工作空间]/需求要求记录.md
cp TEMPLATE_chat.md      [你的工作空间]/chat.md
cp TEMPLATE_工作空间宪法.md  [你的工作空间]/docs/references/共享工作空间宪法.md
```

## 第4步：配置 AI 工具

根据你用的 AI 工具（Copilot / Cursor / Reasonix / Claude Code 等），设置：

1. 告诉 AI 先读 `工作规范.md` 和 `memory.md`
2. 让 AI 以 `[您好/你好]，[你的名字]，[AI名称] 就位` 开头回复
3. 操作完成后让 AI 同步到 `chat.md`

## 第5步：开始第一个协作

```bash
# 在 chat.md 写你的第一个需求
[你的名字]: [MM-DD HH:MM] 我需要...

# AI 会：
# 1. 打招呼 + 对时
# 2. 读取工作规范.md 和 memory.md
# 3. 分析你的需求
# 4. 执行操作 → 更新 chat.md
```

## 规则熟悉期

| 天数 | 建议 |
|:---:|------|
| 第1-3天 | 每天读一遍 `工作规范.md` 的22条规则 |
| 第4-7天 | 重点关注红线规则：规则10（打招呼）、规则8（同步）、规则20（Git） |
| 第2周 | 检查违规记录，看看哪些规则容易踩 |
| 第3周+ | 调整规则到你团队的实际情况 |

---

*有问题随时调整规则。规则 15 规定：规则新增权归用户。这套模板是你的起点，不是终点。*
