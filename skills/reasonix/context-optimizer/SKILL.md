# Context-Optimizer — 上下文自动精简

每次对话开始时自动检查并优化上下文，防止 token 膨胀。

## 检查项

### 1. 记忆数量
- 检查 `memory list` 结果
- 记忆 > 5 条 → 建议合并（同类合并，信息不丢失）
- 检测过时记忆 → 提示更新

### 2. chat.md 行数
- 读取 `D:\Reasonix_Workspace\chat.md`
- ≤ 500 行 → 无需处理
- 501~1000 行 → 建议归档
- > 1000 行 → 强制归档（规则第14条）

### 3. 第零条执行
- 同一次对话中，已读取且未变更的7个核心文件 → 不反复重读
- 仅在对话开始或文件变更时读取

### 4. 输出模式
- 确认 token-budget-advisor 已激活（默认开启）
- 如用户未选深度级别，每次回复前展示4档选择

## 执行时机
- 每次对话开始：自动检查1+2+3+4
- 每10轮对话：复查记忆是否可合并
- chat.md > 1000行：立即触发，不等待

## 输出格式

```
## 📊 上下文体检

| 检查项 | 状态 | 详情 |
|--------|------|------|
| 记忆数量 | ✅/⚠️ | X条，建议合并至Y条 |
| chat.md | ✅/⚠️/🚨 | X行 |
| 文件重读 | ✅ | 同会话不反复读 |
| 输出控制 | ✅ | token-budget-advisor已激活 |

**建议：**（如有）
```

## 规则依据
- 第零条：防重复读取
- 第12条：防冗余、防过大
- 第14条：chat.md超千行自动归档

## 用法指南

**一句话：** 每次对话开始自动检查上下文健康度（记忆/chat.md/重读/输出），防 token 膨胀。

**调用方式：** 对话开始时自动执行，也可手动 `/context-optimizer`

**示例：**
```
/context-optimizer
→ 
## 📊 上下文体检
| 检查项 | 状态 | 详情 |
|--------|------|------|
| 记忆数量 | ✅ | 3条，健康 |
| chat.md | ⚠️ | 620行，建议归档 |
| 文件重读 | ✅ | 同会话不反复读 |
| 输出控制 | ✅ | token-budget-advisor已激活 |

**建议：** chat.md 超500行，执行 /chat-archive
```

**输入：** 无（自动检测）  
**输出：** 4项体检表格 + 建议
