# Op-Log — 操作记录模板生成

用于按规则第 9 条生成操作记录文件。

## 文件命名

`[操作名称]_YYYY-MM-DD.md` — 操作名称用中文概括本次对话主题

## 必填字段

每条记录必须包含：

| 字段 | 说明 |
|------|------|
| 序号 | 自增数字 |
| 时间 | HH:MM 格式 |
| 执行者 | Reasonix / Copilot / Claude |
| 类型 | 启动/停止/配置/修改/核查/创建/部署/SSH/时间戳/同步/记忆/诊断 |
| 摘要 | 一句话描述做了什么 |
| 作用 | 该操作的目的和效果 |
| Shell命令 | 实际执行的命令（文件编辑类标注"文件编辑"即可） |

## 文件模板

```markdown
# [操作名称] 操作记录

> 日期：YYYY-MM-DD
> 执行者：Reasonix / Copilot / Claude

| # | 时间 | 执行者 | 类型 | 摘要 | 作用 | Shell 命令 |
|---|------|--------|------|------|------|-----------|
| 1 | HH:MM | - | - | - | - | - |
```

## 使用方式

每次对话结束后，把执行过的操作追加到对应的操作记录文件。同月内操作追加到同一个文件。

## 已存在的操作记录文件

检查 `D:\Reasonix_Workspace\` 下已有文件，按日期匹配追加或新建。

## 注意

- 操作记录文件也需同步告知到 chat.md（第 8 条）
- 文件超过 1000 行需拆分（第 12 条）

## 用法指南

**一句话：** 按规则第 9 条格式生成操作记录文件，自动编号+时间+类型+命令列。

**调用方式：** `/op-log <本次对话的操作摘要>`

**示例：**
```
/op-log 创建了3个skill，安装了collab-cli，同步了chat.md
/op-log 修复MySQL连接超时：修改bootstrap.yml + 重启容器
```

**输入：** 一句话描述本次对话做了什么操作  
**输出：** 自动生成带序号的表格行，追加到当月操作记录文件（如 `Skill创建与安装_2026-06-14.md`）

**输出格式：**
| # | 时间 | 执行者 | 类型 | 摘要 | 作用 | Shell命令 |
|---|------|--------|------|------|------|-----------|
| N | HH:MM | Reasonix | 创建 | 安装XX skill | 用于XX | `install_skill(...)` |

**自动行为：** 
- 同类操作按月合并到同一文件
- 文件超 1000 行自动提示拆分
- 操作后提醒同步 chat.md（规则第 8 条）
