# learn-topic

研究一个技术或知识主题，输出结构化笔记。

## 流程

1. **搜索** — 用 web_search 搜 3-5 个高质量来源
2. **抓取** — 用 web_fetch 读 2-3 篇最有价值的文章
3. **整理** — 按以下结构输出：

```markdown
# [主题名称]

## 一句话总结
[≤50字]

## 核心概念
- 概念1: 解释
- 概念2: 解释

## 关键细节
[3-5个要点，每个≤3行]

## 常见误区
[如果有]

## 实践建议
[2-3条可操作的下一步]

## 参考来源
- [标题](URL)
```

## 规则
- 优先中文来源，但英文技术文档更准确时可保留
- 控制总输出在 500 字以内
- 标注信息的确定程度（确定/可能/待验证）

## 用法指南

**一句话：** 快速研究一个主题（技术/知识），自动搜索→抓取→整理→输出结构化笔记。子代理模式，不占主对话 token。

**调用方式：** `/learn-topic <主题>`

**示例：**
```
/learn-topic Docker 多阶段构建最佳实践
/learn-topic MySQL 8.0 索引优化策略
/learn-topic Spring Boot 3 新特性
```

**输入：** 任意技术/知识主题（中文或英文）  
**输出：** 结构化笔记——
- 一句话总结（≤50字）
- 核心概念
- 关键细节（3-5要点）
- 常见误区（如有）
- 实践建议（2-3条可操作下一步）
- 参考来源（含 URL）

**特点：** 
- 🔹 子代理执行，不占主对话 token
- 优先中文来源
- 总输出约 500 字
- 标注信息可靠度（确定/可能/待验证）
