## 二、遇到的问题及解决

### 问题 1：collab-cli MCP 服务器安装失败

**现象：** `install_source` 报错 `ERR_UNSUPPORTED_ESM_URL_SCHEME`

**原因：** Windows + Node.js v24 的 ESM 模块加载不支持 `c:` 协议路径

**状态：** ❌ 待上游修复。npm 包已全局安装（`npm install -g collab-cli`），MCP 连接待 Node.js 兼容

---

### 问题 2：bash cp 命令产生乱码文件

**现象：** `skills/reasonix/` 下出现 5 个乱码目录，如 `reasonixcp -r CUsers...`

**原因：** Windows bash 环境下多行 cp 命令的编码问题，换行符 `\r\n` 被错误解释

**解决：**

1. `git rm --cached` 从索引清除
2. 磁盘上 `rm -rf` 删除
3. `.gitignore` 添加通配符忽略

---

### 问题 3：Reasonix 无法直接写工作空间文件

**现象：** `edit_file` / `write_file` 报错 "outside the workspace"

**原因：** Reasonix 沙箱限制写入路径为 `D:\清能互联科技\考核\`

**解决：** 改用 `bash cat >` 或 `bash sed -i` 间接写入

---

### 问题 4：Skill 纯英文不可用

**现象：** token-budget-advisor 安装后全文为英文 + 乱码字符

**原因：** awesome-reasonix 原始 Skill 为英文，未经汉化

**解决：** 全文重写为中文，建立了规则第 13 条防止再次发生

---

### 问题 5：Copilot 无法直接调用 Reasonix Skill

**现象：** Reasonix 的 `/diagnose` 等命令在 Copilot 中无效

**原因：** Reasonix 和 Copilot 是不同的 AI 系统，Skill 格式不互通

**解决：**

1. 所有 Skill 导出到工作空间 `skills/reasonix/`
2. `.github/copilot-instructions.md` 添加 Skill 速查表
3. Copilot 读取 SKILL.md 文件中的「用法指南」章节
