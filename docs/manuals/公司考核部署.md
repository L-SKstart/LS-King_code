## （5）数据库连接常见问题处理

> **应用启动或运行中遇到以下四种数据库报错时，按对应步骤处理。**

---

### 报错一：`Access denied for user 'root'@'xxx' (using password: YES)`

> **原因：** 数据库密码错误或不匹配。

①  登录 MySQL
>
> ```sql
> mysql -uroot -p
> ```

②  修改 root 密码
>
> ```sql
> ALTER USER 'root'@'localhost' IDENTIFIED BY '123456';
> ```
>
> 如果需要从其他主机也使用此密码登录：
>
> ```sql
> CREATE USER IF NOT EXISTS 'root'@'%' IDENTIFIED BY '123456';
> ALTER USER 'root'@'%' IDENTIFIED BY '123456';
> GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' WITH GRANT OPTION;
> FLUSH PRIVILEGES;
> ```

③  同步修改 `bootstrap.yml` 中的密码
>
> ```yaml
> spring:
>   datasource:
>     password: 123456   # 与服务端保持一致
> ```

---

### 报错二：`Host is blocked because of many connection errors`

> **原因：** 连接错误次数超过 `max_connect_errors`（默认 10 次），MySQL 自动封锁该主机。

①  登录 MySQL 并立即解除封锁
>
> ```sql
> mysql -uroot -p
> FLUSH HOSTS;
> ```

②  （可选）调大 `max_connect_errors`，防止再次被封锁
> 编辑 MySQL 配置文件 `/etc/my.cnf`，找到或添加：
>
> ```ini
> max_connect_errors=1000
> ```
>
> 保存后重启 MySQL：
>
> ```bash
> /etc/init.d/mysql restart
> ```

---

### 报错三：`Public Key Retrieval is not allowed`

> **原因：** JDBC 连接 URL 缺少 `allowPublicKeyRetrieval=true` 参数（MySQL 8.x 使用 `caching_sha2_password` 插件时需要）。

①  修改 `bootstrap.yml` 中的 JDBC URL，添加参数：
>
> ```yaml
> spring:
>   datasource:
>     url: jdbc:mysql://192.168.5.128:13306/aj_report?characterEncoding=UTF-8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
> ```
>
> **关键：** 在 URL 末尾加上 `&allowPublicKeyRetrieval=true`

---

### 报错四：`Data too long for column 'json_str' at row 1`

> **原因：** `gaea_report_excel` 表的 `json_str` 字段为 `text` 类型（最大 64KB），报表 JSON 串超长。

①  登录 MySQL 后执行：
>
> ```sql
> ALTER TABLE gaea_report_excel MODIFY COLUMN `json_str` mediumtext CHARACTER SET utf8 COLLATE utf8_general_ci NULL COMMENT '报表json串';
> ```

②  同步修改初始化 SQL 文件（第 693 行）：
> 将 `db/V1.0.0__init.sql` 中：
>
> ```sql
> `json_str` text ...
> ```
>
> 改为：
>
> ```sql
> `json_str` mediumtext ...
> ```

---

**完成上述所有步骤，则表示所有初始环境均已完成配置 。**

### （3.1）OnlyOffice 容器配置——允许访问私有IP

> **报错现象：** 在线预览/编辑文档时失败，OnlyOffice 日志报错：
>
> ```
> error downloadFile:url=http://192.168.5.128:9095/word/template/onlyoffice/file/...
> DNS lookup 192.168.5.128 is not allowed. Because, It is private IP address.
> ```

①  从容器中拷出配置文件到宿主机（仅首次需要）
>
> ```bash
> docker cp onlyoffice-docservice:/etc/onlyoffice/documentserver/default.json /opt/aj-eport/onlyoffice/default.json
> ```

②  在宿主机上修改配置，将 `allowPrivateIPAddress` 改为 `true`
>
> ```bash
> sed -i 's/"allowPrivateIPAddress": false/"allowPrivateIPAddress": true/' /opt/aj-eport/onlyoffice/default.json
> ```

③  编辑 `docker-compose.yml`，在 `volumes:` 下添加一行挂载配置，一劳永逸：
>
> ```yaml
>     volumes:
>       - /opt/onlyoffice/fonts:/usr/share/fonts/truetype/custom:ro
>       - /opt/aj-eport/onlyoffice/default.json:/etc/onlyoffice/documentserver/default.json:ro   # 新增这行
> ```

④  重建容器使配置生效
>
> ```bash
> cd /opt/aj-eport/onlyoffice
> docker-compose down
> docker-compose up -d
> ```

⑤  验证是否生效
>
> ```bash
> docker exec onlyoffice-docservice grep "allowPrivateIPAddress" /etc/onlyoffice/documentserver/default.json
> # 应输出: "allowPrivateIPAddress": true
> ```

> **原理：** OnlyOffice 8.x 的 `request-filtering-agent` 默认禁止访问私有 IP（`192.168.x.x`、`10.x.x.x`、`172.16-31.x.x`），导致无法从 Java 后端下载文档。通过 volume 挂载宿主机修改后的配置文件，即使容器重建也能自动生效。
